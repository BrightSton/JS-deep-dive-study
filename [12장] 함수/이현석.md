- 실행 시점을 개발자가 결정할 수 있고 몇 번이든 재사용이 가능한데, **코드의 재사용**이라는 측면에서 매우 유용.

- 코드의 중복 억제 및 재사용성을 높이는 함수는 **유지보수의 편의성**을 높이고 실수를 줄여 **코드의 신뢰성**을 높임.

- **객체 타입의 값**으로 이름을 붙일 수 있고 함수 이름은 **함수 자신의 역할을 잘 설명**해야 함.

## 12.3 함수 리터럴

- **객체 타입의 값**이기에, 숫자 값을 숫자 리터럴로 생성하고 객체를 객체 리터럴로 생성하는 것처럼 함수도 **함수 리터럴로 생성.**

  - **함수 리터럴**은 function 키워드, 함수 이름, 매개변수 목록, 함수 몸체로 구성.
 
  ```jsx

  }
  ```

- 리터럴은 값을 생성하기 위한 표기법으로 **함수 리터럴**도 값을 생성하며, 이 값이 객체로 **함수는 객체다.**

  - 함수는 객체이면서, 일반 객체와 차별성을 가지는데,

  - 일반 객체는 호출할 수 없지만 **함수는 호출 가능**.

### 12.4.1 함수 선언문

-  함수 리터럴과 형태가 동일하나, **함수 이름 생략 불가능.**


- **함수 선언문은 표현식이 아닌 문**.

  - 함수 선언문을 콘솔에서 실행하면 완료 값 **undefined가 출력.**
 
  - 만약 표현식인 문이라면 표현식이 평가되어 생성된 함수가 출력.

  <div align="center">
    <img src="https://github.com/BrightSton/JS-deep-dive-study/assets/105143449/fb9015c1-372f-49b9-b306-5108d064ee51">
  </div> 
    
- 표현식이 아닌 문은 변수에 할당할 수 없는데,  변수에 할당되는 것처럼 보이는 것이 가능.

  - 함수 선언문은 함수 이름을 생략할 수 없다는 점을 제외하면 함수 리터럴과 형태가 동일.
 
  - 또한, 함수 객체를 생성한다는 점에서 동일하지만 **호출에 차이.**

  - 자바스크립트 엔진이 코드의 문맥에 따라 해석의 차이 존재. 

    - **동일한 함수 리터럴을 표현식이 아닌 문인 함수 선언문으로 해석하는 경우.**
   
      - 자바스크립트 엔진은 생성된 함수를 호출하기 위해 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성.
      
      - 거기에 함수 객체를 할당하기 떄문에 별도의 함수 객체를 가르키는 식별자가 없어도 참조 가능.
     
      - 즉, 함수 이름으로 호출하는 것이 아니라 함수 객체를 가리키는 식별자로 호출.
     
    <div align="center">
      <img src="https://github.com/BrightSton/JS-deep-dive-study/assets/105143449/c7854933-dfde-4524-abd2-f0adc06ba8f5">
    </div>
    <br>

    - **표현식인 문인 함수 리터럴 표현식으로 해석하는 경우.**

      -  함수 외부에서 함수 이름으로 참조할 수 없으므로 함수 이름으로 함수를 호출할 수 없다는 의미.
     
      -  함수를 가리키는 식별자가 없다는 것과 마찬가지.

    ```jsx
    // 기명 함수 리터럴을 단독으로 사용하면 함수 선언문으로 해석
    // 함수 선언문에서는 함수 이름을 생략할 수 없다.
    function foo() {
      console.log("foo");
    }
    foo(); // foo

    // 함수 리터럴을 피연산자로 사용하면 함수 선언문이 아니라 함수 리터럴 표현식으로 해석
    // 함수 리터럴에서는 함수 이름을 생략할 수 있다.
    (function bar() { console.log("bar"); });
    bar(); // ReferenceError: bar is not defined;
    ```

### 12.4.2 함수 표현식

- 값처럼 변수에 할당 가능, 프로퍼티 값도 가능, 배열의 요소도 가능, 일한 성질의 객체가 일급 객체.

 - 따라서, **자바스크립트의 함수는 일급 객체에 해당.**

- 함수 리터럴로 생성한 함수 객체를 변수에 할당 가능하고 이러한 정의 방식이 **함수 표현식.**

 - 함수 리터럴의 함수 이름은 생략 가능하고 이를 **익명 함수**라 명명.

```jsx
// 함수 표현식 
var add = function (x, y) { 
		return x + y;
	}；

console.log(add(2, 5)); // 7
```

- 함수 호출에는 함수 객체를 가리키는 **식별자를 사용**하고, 함수 이름은 함수 내부에서만 유효한 식별자이므로 **외부에서 함수 호출 불가능.**

```jsx
// 기명 함수 표현식
var add = function foo (x, y) {
 		return x + y;
 	}；

// 함수 객체를 가리키는 식별자로 호출 
console.log(add(2, 5)); // 7

// 함수 이름으로 호출하면 ReferenceError가 발생한다.
// 함수 이름은 함수 몸체 내부에서만 유효한 식별자다. 
console.log(foo(2, 5)); // ReferenceError: foo is not defined
```

### 12.4.3 함수 생성 시점과 호이스팅

```jsx
// 함수 참조
console.dir(add); // f add(x, y)
console.dir(sub); // undefined

// 함수 호출
console.log(add(2, 5)); //7
console.log(sub(2, 5)); // TypeError: sub is nor a function

// 함수 선언문
function add(x, y) {
  return x + y;
}

// 함수 표현식
var sub = function (x, y) {
  return x - y;
};
```

- 함수 선언문으로 함수 정의 시 런타임 이전에 함수 객체가 생성, 자바스크립트 엔진이 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성 후 함수 객체를 할당.

  - 함수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징을 **함수 호이스팅.**

- 함수 표현식은 변수에 함수 리터럴 할당으로 변수 할당문의 값은 할당문이 실행되는 시점인 런타임에 평가.

  - 함수 표현식으로 함수를 정의하면 함수 호이스팅이 발생하는 것이 아니라 **변수 호이스팅이 발생.**

- 함수 호이스팅은 함수를 호출하기 전에 반드시 함수를 선언해야 한다는 당연한 규칙을 무시하기 때문에 함수 선언문 대신 함수 표현식을 사용할 것을 권장.
<br>

### 12.4.4 Function 생성자 함수

-  **Function 생성자 함수**에 매개변수 목록과 함수 몸체를 문자열로 전딜하면서 new 연산자와 함께 호출하면 함수 객체를 생성해서 반환.

  -  new 연산자 없이 호출해도 결과는 동일.

- 함수 선언문이나 함수 표현식으로 생성한 함수와 Function 생성자 함수로 생성한 함수가 **동일하게 동작하지 않음.**

```jsx

 		var a = 10;

 		return x + y + a;

	}())；

console.log(addl(l, 2)); // 13

var add2 = (function () {
		var a = 10;
		return new Function('x1, 1y', 'return x + y + a;');
 	}())；
 
console.Iog(add2(l, 2)); // ReferenceError: a is not defined
```

### 12.4.5 화살표 함수

- ES6에서 도입되어 function 키워드 대신 화살표( =>) 를 사용해 좀 더 간략한 방법으로 함수를 선언.

- 기존의 함수보다 표현만 간략한 것이 아니라 내부 동작 또한 간략화.

- 생성자 함수로 사용할 수 없으며，기존 함수와 this 바인딩 방식이 다르고，prototype 프로퍼티가 없으며, arguments 객체를 생성 불가능.

```jsx
// 화살표 함수
const add = (x, y) => x + y;

console.Iog(add(2, 5)); // 7
```

## 12.5 함수 호출

### 12.5.1 매개변수와 인수

- 실행에 필요한 값을 내부로 전달할 필요가 있는 경우, 매개변수(인자)를 통해 인수를 전달.

```jsx
 // 함수 선언문
function add(x, y) {
	return x + y;
}

// 함수 호출
// 인수 1과 2가 매개변수 x와 y에 순서대로 할당되고 함수 몸체의 문들이 실행. 
var result = add(l, 2);
```

- 인수는 함수를 호출할 때 지정, 개수와 타입에 제한 X.

  - 인수 부족시 인수가 할당되지 않은 매개변수의 값은 undefined.

  - 매개변수보다 인수가 더 많은 경우 초과된 인수는 무시.(암묵적으로 arguments 객체의 프로퍼티로 보관)

  - arguments 객체는 함수를 정의할 때 매개변수 개수를 확정할 수 없는 가변 인자 함수를 구현할 때 유용하게 사용.

```jsx
function add(x, y) {
	return x + y;
 }

console.log(add(2, 5, 10)); // 7

function add(x, y) {
	console.log(arguments);
	// Arguments(3) [2, 5, 10, callee: f, Symbol(Symbol.iterator): f]
 	return x + y;
 }
```

- 함수 호출 시 함수 내에서 암묵적으로 매개변수가 생성, 일반 변수와 동일하게 undefined로 초기화된 이후 인수가 순서대로 할당.

  - 매개변수는 함수 내부에서만 참조 가능, 함수 외부에서는 참조 불가능.

### 12.5.2 인수 확인

- 코드상으로는 어떤 타입의 인수를 전달해야 하는지，어떤 타입의 값을 반환하는지 명확하지 않음.

- 자바스크립트의 경우 함수를 정의할 때 적절한 인수가 전달되었는지 확인 필요.

```jsx
function add(x, y) {
	if (typeof x !== 'number' !| typeof y !== 'number') { 

	// 매개변수를 통해 전달된 인수의 타입이 부적절한 경우 에러를 발생시킨다. 
	throw new TypeError('인수는 모두 숫자 값이어야 합니다. '); 
	}
 	return x + y;
 }

console.log(add(2));         // TypeError: 인수는 모두 숫자 값이어야 합니다.
console.log(add( 'a', 'b')); // TypeError: 인수는 모두 숫자 값이어야 합니다.
```

  - 함수 내부에서 적절한 인수가 전달되었는지 확인함에도 부적절한 호출을 사전에 방지할 수 없고 에러는 런타임에 발생.

  - 타입스크립트와 같은 정적 타입 선언이 가능한 자바스크립트의 상위 확장 도입을 통해 컴파일 시점에 부적절한 호출을 방지 가능.

  - 인수가 전달되지 않은 경우 단축 평가를 사용해 매개변수에 기본값 할당으로 디버깅 가능.

  - ES6에서 도입된 매개변수 기본값 사용으로 함수 내에서 수행하던 인수 체크 및 초기화를 간소화 가능.

  ```jsx
  ES6 이전 단축 평가.
  function add(a, b, c) {
 		 a = a || 0;
 		 b = b || 0;
 		 c = c || 0;
 		 return a + b + c; 
  }

  ES6 이후 매개변수 기본값 사용.
  function add(a = 0, b = 0, c = 0) {
 		 return a + b + c;
  }

  console.log(add(l, 2, 3)); // 6
  console.log(add(l, 2)); // 3
  console.1og(add(l)); // 1
  console.log(add()); // 0
  ```

### 12.5.3 매개변수의 최대 개수

- ECMAScript 사양에서는 매개변수의 최대 개수에 대해 명시적으로 제한 X.

- 매개변수는 순서에 의미가 있는데, 많아지면 함수를 호출할 때 전달해야 할 인수의 순서도 고려할 필요 생김.

  - 함수 사용법을 이해하기 어렵게 만들고 실수를 발생시킬 가능성 높임.

  - 매개변수의 개수나 순서가 변경됨에 따라 호출 방법도 바뀌게 되고 함수를 사용하는 코드 전체에 영향.

  - 즉, 유지 보수 측면에서 불리함.

- **이상적인 함수는 한 가지 일만 해야 하며 가급적 무겁지 않아야함.**

  -  함수가 여러 가지 일을 하는건 바람직 하지 않으며, 매개변수가 적을수록 유리.

  -  매개변수가 최대 3개 이상을 넘지 않는 것을 권장.

  -  **그 이상의 매개변수가 필요하다면 하나의 매개변수를 선언하고 객체를 인수로 전달하는 것이 유리.**

  - 프로퍼티 키만 정확히 지정하면 매개변수의 순서 상관 X.

  - 명시적으로 인수의 의미를 설명하는 프로퍼티 키를 사용하게 되므로 코드의 가독성 향상.

  - **주의할 것**은 함수 외부에서 내부로 전달한 객체를 내부에서 변경시 함수 외부의 객체가 변경되는 부수 효과가 발생.

 ```jsx
 // jQuery의 ajax 메서드에 객체를 인수로 전달하는 예.
 $.ajax({
 	method: 'POST',
 	url: '/user',
 	data: { id: 1, name: 'Lee' },
 	cache: false
 	})；
 ```

### 12.5.4 반환문

- return 키워드와 표현식(반환값)으로 이뤄진 반환문을 사용해 실행 결과를 함수 외부로 반환 가능.

- 반환문의 역할.

  - 함수의 실행을 중단하고 종료하기에 반환문 이후에 다른 문이 존재하면 그 문은 실행되지 않고 무시.

  - return 키워드 뒤에 오는 표현식을 평가해 반환하므로, 키워드 뒤에 반환값으로 사용할 표현식을 명시적으로 지정하지 않으면 undefined가 반환.

 ```jsx
 function mu1tip1y(x, y) {
 		return x * y; // 반환문

		// 반환문 이후에 다른 문이 존재하면 그 문은 실행되지 않고 무시된다.
 		console.log('실행되지 않는다.');
 }

 console.log(multiply(3, 5)); // 15

 function foo () {
 		return;
 }

 console.log(foo()); // undefined
 ```

- return 키워드와 반환값으로 사용할 표현식 사이에 줄바꿈시 세미콜론 자동 삽입 기능에 의해 의도치 않은 결과가 발생.

- 함수 내부에서만 사용 가능하고, 생략이 가능한데 마지막 문까지 실행한 후 암묵적으로 undefined를 반환.

## 12.6 참조에 의한 전달과 외부 상태의 변경

 ```jsx
// 매개변수 primitive는 원시 값을 받고，매개변수 obj는 객체를 받음.
function changeVal(primitive, obj) {
			primitive += 100;
			obj.name = 'Kim';
		}

// 외부 상태 
var num = 100;
var person = { name: 'Lee' };

console.log(num); // 100
console.log(person); // {name： "Lee"}

// 원시 값은 값 자체가 복사되어 전달, 객체는 참조 값이 복사되어 전달.
changeVal(num, person);

// 원시 값은 원본이 훼손 X.
console.log(num); // 100

// 객체는 원본이 훼손. 
console.log(person); // {name: "Kim"}
 ```

- 매개변수에 원시 타입 인수는 원본 훼손 X, 객체 타입 인수는 원본이 훼손.

- 함수가 외부 상태를 변경하면 상태 변화를 추적하기 어려움.

  - 코드의 복잡성을 증가, 가독성 저하.

  - 해결 방법.

    - 객체의 변경을 추적하는 옵저버 패턴.

    - 객체를 불변 객체로 만들어 사용.

    - 외부 상태를 변경하지 않고 외부 상태에 의존하지도 않는 순수 함수의 사용.

### 12.7.2 재귀 함수

- 자기 자신을 호출하는 것을 재귀 호출이라 하며, 자기 자신을 호출하는 함수 재귀 함수.

- 함수 내부에서는 함수 이름을 사용해 자기 자신을 호출 가능.

- 함수 표현식으로 정의한 함수 내부에서는 함수 이름은 물론 식별자로도 자기 자신을 재귀 호출 가능.

```jsx
// ex) 팩토리얼(계승)은 1부터 자신까지의 모든 양의 정수의 곱.

// n! = 1 * 2 * ... * (n-l) * n 
function factorial(n) {
 		// 탈출 조건: n 이 1 이하일 때 재귀 호출을 범춘다.
 		if (n <= 1) return 1;

		// 재귀 호출
		return n * factorial(n - 1);
}

// 함수 표현식
var factorial = function foo(n) {
 		// 탈출 조건: n 이 1 이하일 때 재귀 호출을 범춘다. 
 		if (n <= l) return 1;

 		// 함수를 가리키는 식별자로 자기 자신을 재귀 호출
 		return n * factorial(n - l);

		// 함수 이름으로 자기 자신을 재귀 호출할 수도 있다.
		//console.log(factorial === foo); // true
 		// return n * foo(n - 1);
}；

// for ... while 문을 이용.
function factorial(n) { 
 		if (n <= l) return 1;
  		var res = n;

 		while (--n) res *= n; 
		return res; 
}
```

### 12.7.3 중첩 함수

- 함수 내부에 정의된 함수를 **내부 함수 or 중첩 함수**, 중첩 함수를 포함하는 함수가 **외부 함수.**
 
  - 외부 함수 내부에서만 중첩 함수 호출 가능.

- ES6 이전, 코드의 최상위 또는 다른 함수 내부에서만 정의 가능.

- ES6 이후, if 문이나 for 문 등의 코드 블록 내에서도 정의 가능.
 
- 호이스팅으로 인한 혼란 발생으로 인해 if 문이나 for 문 등의 코드 블록에서 함수 선언문을 통해 함수 정의는 선호 X.

```jsx
function outer() {
 		var x = 1;

 		// 중첩 함수
 		function inner() {
 		 		var y = 2;

				// 외부 함수의 변수를 참조할 수 있다. 
				console.log(x + y); // 3 
				}
		inner();
		}
outer();
```

### 12.7.4 콜백 함수

- 함수의 매개변수를 통해 다른 함수의 내부로 전달되는 함수를 **콜백 함수.**

- 매개변수를 통해 함수의 외부에서 콜백 함수를 전달받은 함수를 **고차 함수.**

  - 중첩 함수가 외부 함수를 돕는 헬퍼 함수의 역할을 하듯, 콜백 함수도 고차 함수에 전달되어 헬퍼 함수의 역할.

  - 중첩 함수는 고정되어 있어서 교체하기 곤란하지만, 콜백 함수를 자신의 일 부분으로써 합성 가능.

- 콜백 함수가 고차 함수 내부에만 호출된다면 콜백 함수를 익명 함수 리터럴로 정의후 바로 고차 함수에 전달하는 것이 일반적.

  - 함수 외부에서 호출 필요가 있거나 전달받는 함수가 자주 호출된다면, 콜백 함수 정의 후 함수 참조를 고차 함수에 전달하는 편이 효율적.

- 함수형 프로그래밍 패러다임뿐만 아니라 비동기 처리, 배열 고차 함수에서도 사용.

  - 배열은 사용 빈도가 높은 자료구조이고 배열을 다룰 때 배열 고차 함수는 매우 중요.

```jsx
// 콜백 함수를 사용한 이벤트 처리
// myButton 버튼을 클릭하면 콜백 함수를 실행.
document.getElementById("myButton").addEventListener("click", function () {
	console.log("button clicked!");
});

// 콜백 함수를 사용한 비동기 처리
// 1초 후에 메시지를 출력한다.
setTimeout(function () {
	console.log("1초 경과");
}, 1000);

// 콜백 함수를 사용하는 고차 함수 map
var res = [1, 2, 3].map(function (item) { 
	return item * 2;
 })；
console.log(res); // [2, 4, 6]

// 콜백 함수를 사용하는 고차 함수 filter 
res = [1, 2, 3].filter(function (item) { 
	return item % 2;
})；
console.log(res); // [1, 3]

// 콜백 함수를 사용하는 고차 함수 reduce 
res = [1, 2, 3].reduce(functior)(acc, cur) { 
	return acc + cur;
 }, 0)；
console.log(res); // 6
```

### 12.7.5 순수 함수와 비순수 함

- **순수 함수** : 어떤 외부 상태에 의존하지도 않고 변경하지도 않는, 부수 효과가 없는 함수.

  - 동일한 인수가 전달되면 언제나 동일한 값을 반환하는 함수.

  - 매개변수를 통해 함수 내부로 전달된 인수에게만 의존해 값을 생성해 반환.

  - 일반적으로 최소 하나 이상의 인수를 전달 받음.

  - 인수를 변경하지 않는 것이 기본으로 인수의 불변성을 유지.

  ```jsx
  var count = 0; // 현재 카운트를 나타내는 상태

  // 순수 함수 increase는 동일한 인수가 전달되면 언제나 동일한 값을 반환.
  function increase(n) {
 	 return ++n;
  }

  // 순수 함수가 반환한 결과값을 변수에 재할당해서 상태를 변경.
  count = increase(count);
  console.log(count); // 1

  count = increase(count);
  console.log(count); // 2
  ```

- **비순수 함수** : 외부 상태에 의존하거나 외부 상태를 변경하는, 부수 효과가 있는 함수.

  - 외부 상태에 따라 반환값이 달라지므로 외부 상태에 의존하는 함수.

  - 함수의 외부 상태를 변경하는 부수 효과 O.

  - 함수 내부에서 외부 상태를 참조하지 않더라도 매개변수를 통해 객체를 전달 받은 경우도 포함.

  ```jsx
  var count = 0; // 현재 카운트를 나타내는 상태: increase 함수에 의해 변화.

  // 비순수 함수
  function increase() {
  	 return ++count; // 외부 상태에 의존하며 외부 상태를 변경.
  }

  // 비순수 함수는 외부 상태(count)를 변경하£로 상태 변화를 추적하기 어려움. 
  increase();
  console.log(count); // 1

  increase();
  console.log(count); // 2
  ```
